Namespace FlashcardsApp.Entities

Classes

Card

Entity depicting card data. Card can't exist on its own, only within certain deck.

Deck

Entity depicting deck data. Deck has to be initially made by user, but if user gets deleted, and deck was public then it stays in database with empty CreatorId.

RevisionLog

Entity which is the basis of learning process.
All cards encountered by a certain user will be stored here exactly once.
When user encouters certain card again, the log will be updated.
Otherwise it would lead to huge amount of records.
